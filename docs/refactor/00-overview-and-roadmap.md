# VHDX-RS 重构概览与路线图

## 1. 项目概述

VHDX-RS 是一个纯 Rust 实现的 VHDX (Virtual Hard Disk v2) 虚拟磁盘文件格式库，完整支持 Microsoft MS-VHDX 规范。该库提供完整的 VHDX 文件读写能力，支持固定磁盘、动态磁盘和差异磁盘三种类型，并具备崩溃恢复机制。

### 项目定位

- **核心功能**: VHDX 格式解析、验证、读写操作
- **支持格式**: Fixed (固定)、Dynamic (动态)、Differencing (差异)
- **安全保障**: Log 日志系统实现崩溃一致性
- **应用场景**: 虚拟化平台、磁盘镜像管理、数据备份恢复

## 3. 重构领域概览

基于代码审查，识别出以下 7 个关键重构领域：

### 3.1 生命周期 Bug 修复 (LogWriter)

**问题描述**: `LogWriter` 在首次写入后即被消耗，导致后续写入操作失败，严重影响崩溃恢复功能。

**影响范围**: 
- `src/log/writer.rs`
- `src/file/vhdx_file.rs` 中的写入流程

**风险等级**: 高

---

### 3.2 BlockIo 代码去重

**问题描述**: `DynamicBlockIo` 和 `DifferencingBlockIo` 存在约 700 行重复代码，主要是块查找和扇区位图处理逻辑。

**涉及文件**:
- `src/block_io/dynamic.rs`
- `src/block_io/differencing.rs`

**建议方案**: 提取公共 trait 方法或创建共享辅助模块。

**风险等级**: 中

---

### 3.3 魔法数字消除

**问题描述**: 代码中分散 40+ 处硬编码数值，如 `1024 * 1024` (1MB 对齐)、各种偏移量常量等。

**示例**:
```rust
// 当前写法
let aligned = (size + (1024 * 1024 - 1)) & !(1024 * 1024 - 1);

// 期望写法
const MB: u64 = 1024 * 1024;
let aligned = align_up(size, VHDX_ALIGN_MB);
```

**风险等级**: 低

---

### 3.4 复杂函数拆分

**问题描述**: 部分函数过长，逻辑复杂，难以测试和维护。

| 函数 | 当前行数 | 建议拆分 |
|------|----------|----------|
| `VhdxFile::open()` | ~150 行 | 5-6 个辅助函数 |
| `VhdxBuilder::create()` | ~335 行 | 8-10 个阶段函数 |
| `DifferencingBlockIo::read_block()` | ~120 行 | 3-4 个逻辑单元 |

**风险等级**: 中

---

### 3.5 类型安全增强

**问题描述**: 直接使用 `u64` 表示文件偏移量、块索引等，缺乏类型区分，易产生单位混淆错误。

**建议引入 Newtype 模式**:
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct FileOffset(u64);

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct BlockIndex(u64);

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct ChunkIndex(u32);
```

**影响范围**: 全代码库的偏移量相关参数

**风险等级**: 中

---

### 3.6 BlockIo 实例缓存

**问题描述**: 当前每次读写调用都创建新的 `BlockIo` 实例，造成不必要的堆分配和初始化开销。

**当前模式**:
```rust
pub fn read(&mut self, offset: u64, buf: &mut [u8]) -> Result<usize> {
    let mut block_io = self.create_block_io()?; // 每次创建
    block_io.read(offset, buf)
}
```

**建议模式**: 在 `VhdxFile` 初始化时创建并缓存 `BlockIo` 实例。

**风险等级**: 中

---

### 3.7 未使用代码清理

**问题描述**: `BlockCache` 模块 (199 行) 完全未使用，却需要维护。

**涉及文件**:
- `src/block_io/cache.rs` - 完整未使用

**建议**: 删除或实现实际缓存功能。

**风险等级**: 低

---

### 3.8 错误处理统一

**问题描述**: 代码中混用 `Option` 和 `Result`，转换逻辑不一致，错误信息丢失。

**问题模式**:
```rust
// 不一致的处理
let value = some_option?; // 在 Result 函数中使用
let value = some_result.ok()?; // 错误信息被丢弃
```

**建议**: 统一使用 `Result` 类型，明确定义错误转换边界。

**风险等级**: 中
