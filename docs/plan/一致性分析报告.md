## 未被计划定义的行为（plan undefined behaviors）

以下行为在 `API.md` 计划中完全未定义，是实际实现独有的行为。

### 1. 日志内存回放覆盖层 (`ReplayOverlay`)

- **`ReplayOverlay`** + **`ReplayWrite`** 结构体 — `InMemoryOnReadOnly` 策略的完整实现
- `build_replay_overlay()` — 基于日志条目解析写入片段，构建覆盖层
- `apply_replay_overlay()` — 在读取时按文件偏移合并覆盖数据

**位置**: `src/file.rs:94-1206`
**触发条件**: `LogReplayPolicy::InMemoryOnReadOnly` 或 `Auto`（只读模式）

---

### 2. Dynamic 写入的自动块分配

- `get_or_allocate_block()` — 检查 BAT 条目状态，未分配时触发分配
- `allocate_payload_block()` — 在文件末尾 1MiB 对齐处分配新块，更新 BAT 内存缓存 + 持久化到磁盘
- 分配流程：`extend file` → `update BAT cache` → `write BAT to disk` → `sync_all`

**位置**: `src/file.rs:607-692`
**触发条件**: Dynamic 磁盘写入未分配的 payload block

---

### 3. PartiallyPresent 块读取的扇区位图逻辑

- 按 `chunk_ratio` 计算 `sector_bitmap_bat_index`
- 读取位图数据后按位解析
- 根据位图逐扇区判定：bit=1 从子盘 payload 读取，bit=0 回退父链
- 位图缺失时整个范围回退父链

**位置**: `src/file.rs:354-487`
**触发条件**: 差分盘 + `PartiallyPresent` 状态 payload block

---

### 4. 差分盘链式读取

- `open_parent_for_read()` — 按 `relative_path` → `volume_path` → `absolute_win32_path` 顺序解析父路径并打开
- `read_from_parent_chain_cached()` — 父句柄懒加载 + 缓存，按虚拟偏移读取

**位置**: `src/file.rs:227-273`
**触发条件**: 差分盘 (`has_parent=true`) 读取零/未映射/NotPresent 块

---

### 5. 会话头部初始化 (`init_session_header`)

- 可写打开时更新非活动头部副本
- 递增 `sequence_number`（当前 +1）
- 生成新的 `FileWriteGuid`
- 交替写入策略（仅写非活动头部位置）

**位置**: `src/file.rs:984-1026`
**触发条件**: `File::open(path).write().finish()` 或创建完成后重新打开

---

### 6. 创建路径的完整布局计算与构造

- `calculate_layout()` — 计算各区域偏移和大小（BAT 向上 1MiB 对齐、Metadata 向上 1MiB 对齐、Log 固定 1MiB、Payload 向上 1MiB 对齐）
- `create_metadata()` — 构造元数据区域（表头 + 5/6 个表项 + 数据），含 Parent Locator payload 构建
- `create_bat_data()` — 构造 BAT 数据（Fixed 标记 FullyPresent + 连续偏移、Dynamic 全零）
- `create_region_table()` — 构造区域表（BAT + Metadata 两条目，自动 CRC32C）
- `build_parent_locator_payload()` — UTF-16LE 编码构造 Parent Locator
- `validate_create_params()` — 参数验证（大小/块大小/扇区大小约束）

**位置**: `src/file.rs:1280-1953`

---

### 7. Windows 共享模式打开

- `open_file_with_share_mode()` — Windows 上 `FILE_SHARE_READ | FILE_SHARE_WRITE` 共享模式，避免文件被挂载时锁定
- 错误码 5 (`ACCESS_DENIED`) 转为 `Error::FileLocked`

**位置**: `src/file.rs:814-842`

---

### 8. BAT 交错排列逻辑

- `is_sector_bitmap_entry_index()` — 基于 chunk_ratio 判断 BAT 索引是否属于 Sector Bitmap 条目
- `calculate_chunk_ratio()` — 公式: `(2^23 × logical_sector_size) / block_size`
- `calculate_payload_blocks()` — `ceil(virtual_disk_size / block_size)`
- `calculate_sector_bitmap_blocks()` — `ceil(payload_blocks / chunk_ratio)`
- `calculate_total_entries()` — `payload_blocks + sector_bitmap_blocks`

**位置**: `src/sections/bat.rs:125-186`
**公式依据**: MS-VHDX §2.5

### 10. Validation 模块的详细校验逻辑

`SpecValidator` 实现 9 个校验入口，均在计划中仅有方法签名无实现细节：

| 方法 | 校验内容 | 代码行数 |
|------|----------|----------|
| `validate_header()` | 文件类型签名/头部签名/CRC/版本/日志版本 | ~45 行 |
| `validate_region_table()` | 区域签名/CRC/entry_count/零长度/重复GUID/required unknown | ~75 行 |
| `validate_bat()` | 条目数/状态类型/偏移一致性/Fixed约束/PartiallyPresent约束 | ~115 行 |
| `validate_metadata()` | 表头签名/保留字段/entry_count≤2047/零长度/越界/重复GUID/重叠/块大小 | ~175 行 |
| `validate_required_metadata_items()` | 5个必需元数据项存在性 | ~60 行 |
| `validate_log()` | 签名/CRC/GUID一致性/描述符/撕裂/序列号单调性 | ~185 行 |
| `validate_parent_locator()` | locator_type/键名唯一性/parent_linkage/路径键 | ~125 行 |
| `validate_parent_chain()` | 单跳 DataWriteGuid 一致性 | ~85 行 |

**位置**: `src/validation.rs:48-1005`
