# MS-VHDX 延迟加载扩展标准（v20260501）

> 本文定义在 **MS-VHDX v20240423** 基础上的延迟加载实现扩展约束。目标是：明确 `Sections` 级缓存加载、失效与重载语义，确保结构读取在日志回放与写路径更新后保持一致。

---

## 1. 范围与定位

1. 本扩展覆盖 `Sections` 的延迟加载缓存行为：加载、可变访问、缓存失效、强制重载。
2. 本扩展不修改 MS-VHDX 的文件格式定义，不引入新的 on-disk 布局。
3. 本扩展属于实现标准层，不作为对外 API 计划契约。

---

## 2. 术语与一致性要求

### 2.1 术语

- **延迟加载缓存**：`Sections` 在首次访问 section 时从文件读取并驻留内存的对象缓存。
- **强制重载（force reload）**：主动清空缓存后，下一次访问必须重新从文件读取。
- **结构面一致性**：Header/BAT/Metadata/Log 等结构读取结果与文件当前状态一致。

### 2.2 一致性要求

- 本文中的 `MUST / SHOULD / MAY` 继承 RFC2119 语义。
- 若与 MS-VHDX §1.7 + §2 冲突，以 MS-VHDX 规范为准。

---

## 3. `Sections` 延迟加载基线

1. `header()`、`bat()`、`metadata()`、`log()` 在缓存缺失时 **MUST** 按各自 offset/size 从文件读取并解析。
2. 同一 `Sections` 实例内，缓存存在时读取路径 **MAY** 复用缓存以减少重复 IO。
3. 任何缓存复用行为 **MUST NOT** 绕过显式缓存失效语义。

---

## 4. `bat_mut()` 扩展约束

`bat_mut()` 用于写路径获取 BAT 可变缓存引用，约束如下：

1. 首次调用时，若 BAT 缓存不存在，实现 **MUST** 从 BAT 区域加载并解析后再返回。
2. 成功后实现 **MUST** 返回缓存对象的可变借用（`RefMut<Bat>` 语义）用于内存态更新。
3. `bat_mut()` **MUST NOT** 隐式执行 BAT 落盘或 `sync_all`；持久化由上层写路径显式负责。
4. BAT 加载/解析失败时，**MUST** 返回可定位错误。

---

## 5. `invalidate_caches()` 扩展约束

`invalidate_caches()` 用于显式失效缓存，约束如下：

1. 调用后实现 **MUST** 清除 `header`、`bat`、`metadata`、`log` 四类缓存。
2. 失效后首次访问任一 section 时，实现 **MUST** 重新从文件读取并重建缓存（force reload）。
3. 该方法 **MUST** 为幂等：连续调用不应引入额外副作用。

---

## 6. 调用时机与流程约束

以下场景实现 **SHOULD** 调用 `invalidate_caches()`：

1. 日志回放已将结构修改应用到文件后；
2. 外部流程直接改写 BAT/Metadata/Log 区域且当前 `Sections` 仍持有旧缓存时。

推荐顺序（示意）：

1. 结构写入或日志回放完成；
2. 持久化完成并返回成功；
3. 调用 `invalidate_caches()`；
4. 后续读取按需延迟重载。

---

## 7. 一致性保证与非目标

1. 未失效前，读取结果 **MAY** 反映缓存快照。
2. 失效后，后续读取 **MUST** 反映文件当前状态，而非失效前快照。
3. 本标准不定义跨线程并发写入一致性模型；并发控制与同步责任在上层。

---

## 8. 最小合规清单（可打勾）

- [ ] `Sections` 四类 section 均支持延迟加载。
- [ ] `bat_mut()` 在缓存缺失时会先加载再返回可变借用。
- [ ] `bat_mut()` 不隐式落盘。
- [ ] `invalidate_caches()` 清空四类缓存并可幂等调用。
- [ ] 缓存失效后首次访问会强制重载。
